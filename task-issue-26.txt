This test defines `handler_check_data` as a nested function. GCC nested functions use stack trampolines and often require an executable stack, which is a security hardening problem and can break on some toolchains/flags. Move this handler to file scope (e.g., `static` function) and pass state via globals or a small struct referenced from `userdata`.

**Copilot:**
`struct Subscriber` stores `userdata`, and `hub_subscribe` accepts it, but `hub_emit` never exposes it to the handler (and `EventHandler` has no userdata parameter). As a result, the `userdata` argument currently has no functional effect. Either remove `userdata` from the API, or change the callback signature / event struct so handlers can access the per-subscription userdata.

**Copilot:**
`hub_emit` passes a pointer to a stack-allocated `struct Event` into handlers. If any handler stores that pointer beyond the callback, it becomes dangling immediately after `hub_emit` returns (UB).
